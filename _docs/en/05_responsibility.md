## Roles

A single person can take multiple roles.

- **Repository Maintainer**: responsible for maintaining the repository, managing/developing automation tools. Needs to be familiar with Git, the platform, and have coding skills.

- **Reviewer**: responsible for reviewing submissions and archiving designs. Needs to understand the basic principles of accepting submissions and know the basic usage of Git repositories.

If you are not familiar with Git or coding, we'd also like to have the following volunteer roles filled:

- **Seeker**: responsible for finding valuable works on the internet and inviting authors to submit their designs. Needs to have excellent interpersonal skills and a basic understanding of submission formats.

- **Expert**: responsible for evaluating the technical level of works and assisting reviewers in determining whether a work is worth archiving. Needs to be a skilled player in a specific TMC field and be aware of the latest developments in that field.


## Responsibility

This archive is responsible for:

- **Archiving**: preserve designs for long-term access and discoverability.
- **Indexing**: keep metadata and categories clear, accurate, and searchable.
- **Maintaining**: keep records current; apply updates while preserving original content.


The repository maintainer is responsible for:

- Auditing the dataset content regularly to ensure the file structure and categories are consistent.
- Reviewing and merging pull requests with reviewers.


The reviewers is responsible for:

- Reviewing submissions according to the acceptance criteria and providing feedback to submitters if necessary.
- Archiving accepted submissions in the correct category.
- Archive works "as-is" when archiving; do not modify the content of the works.
- Synchronizing updates to archived designs when necessary.
- Define and adjust review standards according to submission counts for each category or contraption.
